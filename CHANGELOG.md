# VibeBox Changelog

All notable changes to VibeBox will be documented in this file.

## 2025-11-05

This update focuses on improving testing infrastructure and development workflows to ensure a more reliable and stable platform.

- Added self-hosted Happy Server infrastructure for comprehensive end-to-end testing
- Improved development documentation with upstream directory reference guide
- Added OAuth troubleshooting guide to help resolve authentication issues faster
- Enhanced AI-powered development workflow with context-first skill system
- Improved code quality and maintainability with skill-curator agent

## 2025-10-29

This release brings a completely redesigned user interface with a modern tab-based navigation system, making it easier to manage your coding sessions and boxes.

- Redesigned app interface with new Chats/Boxes/Me tab structure for better navigation
- Added automated E2E testing workflow to ensure reliability and quality
- Updated payment system to support multi-platform subscriptions (iOS, Android, Web)
- Improved documentation with resource pool architecture details
- Enhanced design documentation with Boxes branding alignment

## 2025-10-27

This update introduces critical improvements to authentication and user experience design, with enhanced auto-login functionality.

- Added required Happy auto-login with error blocking UI for better security
- Updated core user experience design to version 2.0 with refined workflows
- Improved page architecture with complete design documentation
- Enhanced China Mobile Cloud provider integration with Portal Gateway support
- Added multi-cloud provider support for flexible deployment options

## 2025-10-23

This release establishes a comprehensive testing infrastructure and improves the overall development methodology with AI-first strategies.

- Added end-to-end testing infrastructure with automated test workflows
- Improved Logto OAuth authentication flow with better redirect handling
- Enhanced documentation with semantic hashtags for easier navigation
- Optimized hashtag system with improved AI agent capabilities
- Added comprehensive Happy Server token lifecycle documentation
- Consolidated E2E tests and improved test coverage

## 2025-10-22

Welcome to the first release of VibeBox! This is a mobile-first AI coding platform built on the Happy open-source project.

This initial release establishes the foundation of VibeBox with essential features for AI-powered mobile coding.

- Added Logto authentication integration for secure user login
- Added Docker Compose development environment for easy local setup
- Implemented complete project structure with Expo client and Next.js server
- Added OAuth authentication support (GitHub, Google)
- Created comprehensive documentation including PRD, white paper, and architecture decisions
- Established project naming and branding (formerly "Happy - Vibe Server", now "VibeBox")
- Added core mobile-first UI with happy-client integration
- Implemented zero-modification solution architecture
- Added complete architecture decision records (ADR 001, ADR 002)

---

**Note:** VibeBox is currently in active development. Each release brings us closer to delivering a seamless mobile-first AI coding experience.
